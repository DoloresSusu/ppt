# 👨‍🏫 演示官 SOP (Demo Lead)


## 一、角色关键职责
*   **演示 (Demo)**：现场演示 AI 工具工作流（Video/Music/Code）。
*   **教学 (Teach)**：确保内容小白友好，降低参与门槛。**从能做到什么产品，到每一个步骤都带着大家走完。**
*   **答疑 (Q&A)**：解答关于工具使用的基础问题。

## 二、核心原则 (必读) ⚠️
1.  **如果是实操导向的活动：不要讲细节！不要讲原理！**:
    - ❌ 错误: "这个 Diffusion 模型的原理是..." "这个参数有 10 种用法..."
    - ✅ 正确: "点这里生成，如果报错就点那里。"
2.  **直接用现成教程**:
    - 如果操作很复杂，**直接播放一个录好的视频**（或者网上找的高质量教程）。让大家自己看，方便暂停回看。
    - 现场只演示**最关键的路径**。
3.  **一般分享不超过 15 分钟**: 你的目标是让大家**开始做**，而不是听你讲课。
4.  **小白视角**：提前询问活动主理人，观众画像是什么，从他们的角度出发思考你要分享的内容。
5.  **保下限 (Minimum Viable Tutorial)**:
    - 教程必须包含一个**最小可行性路径**，跟着走就能做出来，只需改动变量就能出新作品。



## 三、演示内容结构 (标准 15 分钟)
1.  **What (2min)**: 我们今天要玩什么工具？(展示一个酷炫的成品效果)。
2.  **How (5min - 关键步骤)**:
    - 打开哪个网址？（标明是否需要魔法）
    - 点哪里注册？
    - 输入什么 Prompt？
    - 点击生成。
3.  **Resource (3min)**:
    - "详细教程在这里 (展示文档二维码)。"
    - "遇到 Bug 找谁？(指一下技术支持志愿者)。"
4.  **Teaming (5min - 组队指引)**:
    - "怎么组队？(2-4人一组)"
    - "队伍怎么分工？(共学文档第一部分应包含分工表格，e.g. 作词、生成图片、剪辑)。"



## 四、工作流程清单

### 📅 活动前
- [ ] **共学文档**: 准备一个飞书/Notion 文档，包含：
    - 工具链接 (标注是否需要魔法)
    - 账号注册指引
    - **视频教程链接** (B站/YouTube)
    - Prompt 抄作业模板
    - **分工建议表** (让小白知道自己能做什么)
- [ ] **演练 (重要)**:
    - 最好在周四前定下来内容。
    - **与主持人彩排 (5min)**，确认演示时间与重点。
- [ ] **志愿者培训**: 建议请所有志愿者提前学习文档，测试一遍工作流，以便现场协助。

### ⏰ 活动中
- [ ] **投屏演示**: 确保字体够大 (浏览器缩放 150%)。
- [ ] **带学带练**: 关键步骤停顿，确认大家跟上。
- [ ] **巡场**: 演示完后，走下台去解决大家"注册不了/打不开网页"的第一步卡点。



## 五、常见问题与应急处理

| 问题 | 解决方案 |
|------|---------|
| **演示工具现场崩溃** | 立即切换到预先录制的演示视频；或使用备用工具演示类似功能 |
| **参与者跟不上节奏** | 暂停演示，重复关键步骤；或引导大家稍后参考文档自行操作 |
| **网络问题导致工具无法访问** | 提供离线文档/视频教程；或推荐国内可用的替代工具 |
| **时间超时** | 立即跳过细节讲解，直接展示最终效果并告知"详见文档" |



## 六、演示官锦囊
1.  **视频是最好的老师**: 现场网络可能会挂，现场演示可能会翻车，但**提前录好的视频**永远不会。
2.  **把舞台交给参与者**: 你讲得越少，他们动手的时间就越多。
3.  **环境预警**: 考虑到很多人不会"科学上网"，尽量准备国内可用的替代方案，或者提前告知网络要求。
