# 🎛️ 活动统筹 SOP

**一句话目标**：把活动从筹备到收尾跑通，确保按时交付与现场体验稳定。

> 大型活动建议设置「主理人 + 执行统筹」双岗分工。

## 执行清单（筹备到收尾）

### 活动前（T-14 ~ T-1）
- [ ] 快速浏览 [活动统筹手册](/manual/organizer-handbook)，了解标准流程与资源。
- [ ] 基于 [活动执行手册模板](/manual/activity-template) 产出本场活动手册（包含活动流程、分工）。
- [ ] 明确主题/时间/场地与规模，确认预算与赞助支持。
- [ ] 发布志愿者招募公告（找到主持/中控/技术导师/物料/宣发/摄影负责人，协调他们的时间与任务）。
- [ ] 发布报名页与入群指引，设定 T-3/T-1 提醒节奏。
- [ ] 查看 [AI-PPT 模板样例](https://hackppt.01mvp.com/) 并准备开场流程页。

### 活动当天（12:00 - 18:00）
- [ ] 提前 2 小时到场，检查电源/网络/投影/音响。
- [ ] 与主持/中控/技术导师对齐时间轴、展示时长与投票规则。
- [ ] 确认签到、入群与交通指引到位。
- [ ] 关键节点报时与巡场协调，必要时压缩低优先级环节。
- [ ] 路演前确认作品可播放（本地下载/U 盘备份）。

### 活动后（24 小时内）
- [ ] 组织合影、清场复原、志愿者致谢。
- [ ] 收集数据与复盘点，更新 SOP 与教程。

## 注意事项与对齐

### 关键标准
- 先保障“能开始/能交付”，再优化体验。
- 无群=无流程：确保报名者全部入群。
- 统筹只抓关键节奏，不陷入细节执行。

### 常见风险与处理
| 问题 | 处理方式 |
| --- | --- |
| **关键岗位缺席** | 立即调度替补或亲自顶上 |
| **流程超时** | 砍掉低优先级环节或压缩时长 |
| **路演卡顿** | 统一下载/备份，必要时只播高光 |
| **入群遗漏** | 现场补扫 + 活动前逐一私信提醒 |
