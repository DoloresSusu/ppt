# 🎤 主持人 SOP

**一句话目标**：控场节奏、带动氛围，让参与者在清晰流程中完成产出。

## 现场执行清单（活动当天）

### 活动前（T-3 ~ T-1）
- [ ] 与主理人确认活动主题、流程表与关键时间点。
- [ ] 对齐组队方式（自由/抽卡/solo）与关键词/台词限制玩法。
- [ ] 与技术导师 Mentor、分享嘉宾做 5 分钟彩排，明确控时与展示规则。
- [ ] 准备串场词与过渡话术，确认场地关键信息（洗手间/WiFi/电源）。
- [ ] 预设奖项与 1 句话颁奖词，确保更多团队有上台机会。

### 到场准备（12:00 - 13:30）
- [ ] 与中控对齐计时信号与报时节点。
- [ ] 测试麦克风、音响与投屏，确认 PPT 可正常播放。
- [ ] 与物料/场地确认投票道具、奖品、合影位置。
- [ ] 确认投票二维码展示方式与倒计时道具。

### 开场与组队（14:00 - 14:45）
- [ ] 5 分钟内讲清规则与时间轴（组队截止/作品提交/展示时长/投票方式）。
- [ ] 用 PPT 明确活动流程与提交格式（视频类MP4 优先、避免 MOV）。
- [ ] 控制灵感发布时长（1 分钟/人），及时兜底落单人员。
- [ ] 宣布组队结束，进入开发环节。

### 开发与报时（14:45 - 17:00）
- [ ] 按节点报时（30/20/10/5 分钟 + 最后 10 秒倒计时）。
- [ ] 提醒作品提交方式与截止时间，强调提交后尽快确认可播放。

### 展示与收尾（17:00 - 18:00）
- [ ] 说明展示规则与投票方式，保持节奏流畅。
- [ ] 控制每组展示时长（建议 60-90 秒），过长作品仅放高光片段。
- [ ] 每组播完给 15-20 秒投票窗口，并邀请 1 句创作初衷分享。
- [ ] 作品过半时安排 3-5 分钟中场休息，避免审美疲劳。
- [ ] 颁奖与致谢：配乐烘托氛围，邀请志愿者上台并夸到细节。
- [ ] 合影、宣发二维码提醒、清场提示（前排蹲坐/社区旗帜入镜）。

## 注意事项与应急

### 关键标准
- 开场简短有力，避免“先聊 10 分钟再进入主题”。
- 超时先压缩，再跳过，保持整体节奏。
- 展示时长与投票规则必须提前讲清。

::: tip 大型活动可拆分
可安排主主持 + 副主持（台下引导/收集问题/组队兜底），减轻控场压力。
:::

### 沟通协作
| 需要配合 | 对接人 |
| --- | --- |
| 现场设备/投屏/音乐 | [现场中控](./operator) |
| 工具/提交卡点 | [技术导师 Mentor](./tech-support) |
| 现场素材与露出 | [宣发推送](./pr) |
| 合影与关键镜头 | [摄影](./photo) |

### 常见问题与应急处理
| 问题 | 解决方案 |
| --- | --- |
| **冷场** | 主动点名互动，抛出简单问题，带头鼓掌 |
| **超时** | 眼神提示 + 近身提醒，必要时直接进入下一环节 |
| **组队混乱** | 暂停灵感发布，按技能强制配对 |
| **设备故障** | 稳住气氛，先引导短休或互动，等待技术支援 |
| **观众疲劳** | 缩短展示时长，安排中场休息 |
