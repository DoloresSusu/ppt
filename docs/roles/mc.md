# 🎤 主持人工作 SOP

> **适用场景**: 迷你黑客松、创客松、AI工具交流会
> **核心定位**: 这里的"气氛组"组长，活动的"时间守护者"。
> **📝 使用提示**: 请将下方清单视为 Checklist，完成一项勾选一项。



## 一、角色关键职责
*   **控场 (Control)**：把控活动节奏，确保按时进行。**向每个环节对接人约定时间，确保遵守。**
*   **氛围 (Vibe)**：营造轻松、开放、鼓励创造的氛围。
*   **连接 (Connect)**：让参与者感到被看见，引导破冰与互动。
*   **协作 (Collaborate)**：与计时员配合截点，与主理人配合应急。**听分享时建议与电脑控制、计时员站在一起，方便沟通。**



## 二、活动前准备清单

### 📅 活动前
- [ ] **沟通**: 确认活动主题、流程表、分享嘉宾信息。
- [ ] **嘉宾演练**: **务必与分享嘉宾提前演练（5min）**，确认内容重点和时长，告知会严格控时。
- [ ] **共学演练**: 与共学引导员/讲师演练，确保其能带领大家完成"最基本的产品"（MVP）。
- [ ] **准备**: 熟悉场地（洗手间/WiFi/电源），准备串场词。
- [ ] **服装**: 建议穿着社区文化衫或整洁得体服装。

### ⏰ 活动当天 (提前 1-2 小时到场)
- [ ] **对表**: 与主理人确认最终流程，与计时员确认"举牌/响铃"暗号。
- [ ] **测试**: 测试麦克风音量、音响效果、PPT 是否正常播放。
- [ ] **颁奖准备**: 提前与技术支持确认颁奖 PPT 和上台顺序。
- [ ] **热身**: 提前 10 分钟喝水开嗓，调整状态，微笑迎接早到者。



## 三、迷你黑客松主持流程 (标准 4 小时)

### 1️⃣ 签到与开场 (14:00 - 14:30)
*   **14:00 暖场**: 播放轻音乐，微笑迎接，观察签到进度。
    *   *应对迟到*: 如果因迟到无法准时开始，利用这 10 分钟让志愿者引导大家**预分组**或收集技能/分工信息。
*   **14:15 正式开场 (快速切入)**:
    *   **核心原则**: **最快速切入主题**，不要啰嗦，直接吸引大家聚焦到今天要做的事情上。
    *   **自我介绍**: "我是主持人 [名字]，也是大家的啦啦队长！"
    *   **定调**: "无论你是小白还是大神，今天我们只有一个身份——创造者！"
    *   **强调规则**: "15:00 组队截止，17:00 提交作品，WiFi 密码在群里。"
*   **14:20 社区介绍**: 简述社区，播放 PPT。
*   **14:25 流程说明**: 投屏时间轴，强调 **17:00** 是作品提交截止时间。

### 2️⃣ 自由组队 (14:30 - 14:45) 🔥 *高能环节*
*   **14:30 规则**: "2-4人一队。有 Idea 的上台 Pitch (1分钟/人)，没 Idea 的听完直接抢人！"
*   **14:32 Idea Pitch**:
    *   **控场**: 严格限制每人 1 分钟。超时温柔打断："时间到，私下详聊！"
    *   **即时组队**: 每讲完一个，立即问台下："谁感兴趣？举手！" -> 现场指派："好，你们组队，去填队名！"
*   **14:42 兜底**: 统计落单人员，主持人介入强行组队（按技能互补）。
*   **14:45 宣布**: "组队结束，开发正式开始！"

### 3️⃣ 动手实践 & 共学 (14:45 - 17:00)
*   **嘉宾分享/共学环节**:
    *   **保下限**: 确保讲师能**带着大家一步步操作**，达成最基本的产品（MVP）。
    *   **防拖延**: 提醒大家先做完，再完美。
*   **角色**: 转变为"服务者"和"报时鸟"。
*   **报时节点 (必做)**:
    *   **15:00**: "组队最后确认，之后不能换队。"
    *   **16:00**: "已过去 1 小时，加油！"
    *   **16:30**: "最后 30 分钟！未完成的开始收尾！"
    *   **16:50**: "还有 10 分钟，**所有队伍现在开始提交作品！**"
    *   **17:00**: "时间到！停手！"

### 4️⃣ 作品展示 (17:00 - 17:30)
*   **模式选择**:
    *   **摆摊式路演 (强烈推荐，队伍 > 7-8 支)**:
        *   优势：互动更深，即时反馈，气氛更好。
        *   操作：团队在各自座位展示，评委/观众自由流动体验。
    *   **传统路演 (队伍 ≤ 7-8 支)**:
        *   操作：逐个上台展示，每队 3 分钟。
*   **投票方式**:
    *   **无奖金活动**: 自由投票 (如每人 3 票，投给其他团队)。
    *   **有奖金活动**: 评委评选 (由评委评出最佳产品)。
*   **16:55 引导**: "大家回到座位，队长准备展示，每人领 3 个筹码 (如采用自由投票)。"
*   **17:00 开始展示**:
    *   **摆摊式**: 主持人带头巡场，引导人流去冷门摊位；提醒大家多交流、多体验。
    *   **上台式**: 观察时间，超时提示技术人员**调低音量**作为背景音，而不是直接暂停，避免冷场。
*   **17:25 截票**: "最后 5 分钟投票！" -> "投票结束，请计票员统计。"

### 5️⃣ 颁奖与致谢 (17:30 - 18:00)
*   **17:30 等待间隙**: 邀请 2-3 位参与者分享感受（每人 30 秒），填补计票空白。
*   **17:45 颁奖**:
    *   **准备**: 提前确认好 PPT 和奖品。
    *   从第三名报到第一名，制造悬念。
    *   提醒获奖团队发表感言（限时）。
*   **17:55 💖 志愿者致谢 (高光时刻)**:
    *   **话术**: "今天的活动之所以顺利，是因为有一群无私奉献的小伙伴。请所有志愿者上台！"
    *   **流程**: 邀请**主理人/鼓励师**上台，为每位志愿者颁发奖状/纪念品。
    *   **夸夸**: 鼓励师需具体夸奖（如"感谢XX一直在跑腿"），全场掌声致谢。
*   **18:00 结束**: 大合影 -> 宣发反馈二维码 -> 提醒带走垃圾。



## 四、常见问题与应急处理

| 问题 | 解决方案 |
|------|---------|
| **冷场怎么办？** | 自嘲一下，或者点名面善的参与者/老朋友互动；备几个万能问题："今天最让你惊喜的工具是什么？" |
| **超时怎么办？** | 眼神暗示 -> 走近分享者 -> 举麦克风："感谢精彩分享，因为时间关系，我们私下交流！" |
| **设备故障？** | 淡定！"看来 AI 也需要休息一下"，引导大家先进行口头交流或休息，等待技术支援 |
| **组队混乱/落单人多** | 暂停 Pitch，直接按技能分组："所有程序举手 -> 所有设计举手"，强制配对 |
| **投票/计票出错** | 立即请示主理人，启用备用方案（如并列名次、评委追加评选） |



## 五、主持人锦囊 (Tips)
1.  **能量管理**: 你的能量决定现场氛围。累了就深呼吸，别让疲惫传染给大家。
2.  **眼神交流**: 不要只盯着 PPT，多看台下，让每个人感觉你在和他/她说话。
3.  **应急台词**: 准备几句万能救场话术，如"这个环节特别精彩，让我们继续..."
