# 📢 宣发推送工作 SOP


## 一、常用模板与规范 (建议收藏)
*   **公众号发布规范**: [点击查看](https://hackathonweekly.feishu.cn/wiki/A1lawCvxVipWBtkgbqXcqh2Ynxh)
*   **直播指南 (视频号/小红书)**: [点击查看](https://hackathonweekly.feishu.cn/wiki/FFmiwixIAidyL4ksimicc4NVnld)
*   **活动回顾推文模板**:
    > **标题**: 回顾 | [活动主题]：X位创造者，4小时，诞生了Y个AI应用！
    > **结构**:
    > 1.  **高光数据**: 多少人参与，产出多少作品。
    > 2.  **现场直击**: 3-5 张精选照片（专注、讨论、大笑）。
    > 3.  **作品展示**: 挑选 3 个优秀作品，放截图/视频 + 简介。
    > 4.  **PPT/文档下载**: 放上共学文档链接。
    > 5.  **预告**: 下期活动时间。



## 二、工作流程清单

### 📅 活动发布期 (T-7 ~ T-5)
- [ ] **海报制作**:
    - [ ] 包含要素：活动主题、时间、地点、报名二维码、Slogan。
    - [ ] 风格统一：保持社区 VI 风格（如黑白极客风或 vibrant 风格）。
- [ ] **推文撰写**:
    - [ ] 标题党（适度）：吸引点击。
    - [ ] 内容清晰：讲清楚"做什么"、"谁能来"、"有什么收获"。
    - [ ] 插入报名小程序/链接。
- [ ] **多渠道分发**: 公众号、微信群、朋友圈、Luma、活动行。

### 🔄 活动预热期 (T-3 ~ T-1)
- [ ] **倒计时**: 在社群发布倒计时海报或文案，提醒未报名者抓紧。
- [ ] **群维护**: 在活动群内发布预热信息（如共学文档链接、交通指引）。

### ⏰ 活动当天
- [ ] **现场素材**: 转发摄影师拍摄的精彩大片到社群/朋友圈，进行"现场直播"。
- [ ] **直播 (可选)**: 如果有条件，按照《直播指南》开启视频号直播，让不能来的人云参与。
- [ ] **朋友圈直播**: 每隔 1 小时发一条朋友圈（签到->开发->路演->合影），保持热度。


### 🏁 活动后 (沉淀)
- [ ] **素材收集**: 找摄影师要照片，找主理人要作品数据。
- [ ] **回顾推文**: 活动结束 24 小时内发布回顾推文（趁热打铁）。



## 三、常见问题与应急处理

| 问题 | 解决方案 |
|------|---------|
| **摄影师照片延迟** | 提前与摄影师约定交付时间（如当晚 22:00 前）；如仍延迟，先用手机拍的照片应急 |
| **宣发内容审核不通过** | 提前了解平台敏感词规则；准备替代文案；必要时调整发布渠道 |
| **活动数据不全** | 提前与作品管理员、主理人确认数据来源；活动中实时记录关键数字 |



## 四、宣发锦囊
1.  **视觉为王**: 一张好海报胜过千言万语。
2.  **利他思维**: 文案多写"参与者能得到什么"，少写"我们要办什么"。
3.  **素材前置**: 不要等活动结束才想起来要照片，活动开始前就告诉摄影师："我要一张全景、一张特写、一张合影"。
4.  **数据说话**: "大家玩得很开心"不如"4小时产出了 12 个 Demo"有说服力。
