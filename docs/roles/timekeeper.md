# ⏱️ 计时/场控 SOP

::: info 角色核心目标
你是活动的“时间守护者”。你的任务是确保每一个环节都在计划内进行，保护大家的参与体验，防止活动因为超时而导致疲劳或冷场。
:::

计时/场控负责在活动前确认流程表并在现场严格控时，必要时干预超时环节，与主持人和中控紧密配合。

## 一、工作流程清单

### 📅 活动前 (最重要的一步)
- [ ] **确认流程表**:
    - **主动**找主理人/主持人拿到最新版时间表。
    - **核对逻辑**:
        - "这 10 分钟真的够 5 个队伍展示吗？"（如果不合理，直接提出修改建议）。
        - 预留缓冲时间，比如转场、设备调试。
- [ ] **演练与暗号**:
    - **务必与主持人提前演练**，确认提示手势（如：剩 1 分钟举牌，时间到响铃）。
    - 确保主持人知道你的站位，能随时看到你的信号。

### ⏰ 活动流程中 (关键节点)
*   **灵感发布（每人 1 分钟）**:
    - 0:50 -> 举牌 "10秒"。
    - 1:00 -> **果断响铃**。如果对方不停，继续响铃或示意主持人打断。
*   **嘉宾分享**:
    - 按约定时间提醒，超时坚决打断（或通知主持人打断）。
*   **开发环节**:
    - **15:00**: 提醒组队截止。
    - **16:30 (关键截点)**: **强力介入！** 提醒全场："还有 30 分钟，请大家先跑通最小闭环 (MVP)！"
    - **16:50**: "还有 10 分钟，**所有队伍现在开始提交作品！**"
    - **17:00**: "时间到！停手！"
*   **路演环节 (每队 3-5 分钟)**:
    - 严格执行倒计时，确保最后一个队伍也有时间讲完。
    - 作品播放一分钟左右后，如无特殊说明，可提示电脑操作员调低音量。



## 二、沟通与协作

| 遇到以下情况 | 请联系 |
| :--- | :--- |
| **主讲人超时不退场** | [主持人](./mc) 协助打断 |
| **需要调整后续环节时长** | [活动主理人](/organizer/) |
| **投屏计时器显示故障** | [现场中控](./operator) |

## 三、常见问题与应急处理

| 问题 | 解决方案 |
|------|---------|
| **主持人/嘉宾无视时间提示** | 升级提示强度：举牌→响铃→走近舞台；必要时直接告知主理人介入 |
| **流程严重超时** | 立即与主持人、主理人协商压缩后续环节；优先保障核心环节完成 |
| **设备故障导致延误** | 记录延误时长，建议主理人相应压缩后续环节或延长活动结束时间 |
| **参与者要求"再给几分钟"** | 坚持规则，礼貌但坚定地拒绝；如确有特殊情况，请示主理人决策 |



## 四、场控锦囊
1.  **如果你不狠，大家都没车坐**: 活动严重超时会导致大家错过末班车，所以"冷酷"是对大家最大的负责。
2.  **预警大于打断**: 在时间快到前给出一个温和的视觉信号（举牌），最后才是听觉信号（响铃）。
3.  **辅助**: 看到有队伍还在纠结细节而忽略进度时，善意提醒他们"先完成，再完美"。
