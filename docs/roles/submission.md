# 📁 作品管理员 SOP (Submission Manager)

## 一、角色关键职责
*   **流程专家 (Expert)**：极其熟悉提交工具（飞书表格/GitHub/其他平台），是现场这一环节的"技术权威"。
*   **催收 (Collection)**：在截止时间前，消除所有"我不会交"、"我交不上"的障碍。
*   **完整性 (Integrity)**：确保收上来的东西不仅有个链接，还有截图、简介、演示视频、队长名称、队员名称，方便后续宣发。



## 二、工作流程清单

### 📅 准备阶段 (熟悉工具)
- [ ] **搭建 & 测试提交入口**:
    - 创建在线收集表（推荐飞书多维表格，可视化好）。
    - **必填字段设计**:
        - 队名 (Team Name)
        - 作品名 (Project Name)
        - 一句话简介 (One-liner)
        - **演示链接/视频** (最重要，防止现场演示翻车)
        - 队长微信号
- [ ] **测试流程**: 自己先填一遍，确保权限开启（如：飞书文档是否对外可见）。

### ⏰ 活动中 - 提交保障 (16:30 - 17:00)
- [ ] **扫盲教学**:
    - 16:30 上台或发群公告："提交通道已开启！不会填的举手，我过去教你。"
    - 走到各组，确认他们的作品链接（如代码仓库、设计稿）是否已开启"公开分享权限"。
- [ ] **协助截图/录屏**:
    - 很多队伍只会闷头做，忘了留素材。你要提醒："快截个图！不然投票时大家看不到你们的界面。"
    - 必要时，直接上手帮他们录一个 30秒 的 Demo 视频。
- [ ] **最后通牒**:
    - 17:00 准时封板。核对提交列表数量 vs 现场队伍数量。

### 🗳️ 投票与统计 (17:00 - 17:30)
- [ ] **选票管理**: 分发实体筹码或管理线上投票链接。
- [ ] **快速统计**:
    - 活动节奏很快，必须在 5 分钟内算出排名。
    - 如遇平票，立即请示主理人启用"加时赛"或"并列"规则。



## 三、常见问题与应急处理

| 问题 | 解决方案 |
|------|---------|
| **作品链接打不开** | 现场检查分享权限设置，或让队伍重新生成公开链接 |
| **队伍不会录屏/截图** | 现场演示操作步骤，或直接帮忙录制 30 秒 Demo 视频 |
| **提交系统崩溃** | 启用备用方案：微信接龙、邮件提交、或纸质登记 |
| **有队伍忘记提交** | 17:00 后立即巡场确认，给予 2-3 分钟宽限期（需主理人批准） |
| **投票数据丢失** | 如使用线上投票，提前截图备份；如实体投票，指定专人保管筹码 |



## 四、管理员锦囊
1.  **不要只当表格管理员**: 你是**协助者**。如果有人不会生成 GitHub Page，帮他；如果有人不会剪视频，让他直接交个 PPT。
2.  **权限！权限！权限！**: 90% 的演示翻车都是因为"在这个电脑上打不开链接"。一定要在收集表里检查这一点。
3.  **数据归档**: 活动结束后，把这个表格整理好，发给**宣发组**。这是社区最宝贵的资产。
